NEPA

Neuromorphic Event Processing Architecture

The Challenge

Infrastructure inspection requires real-time crack detection and control decisions. Cloud latency, GPU power draw, and bandwidth costs make traditional video processing unsuitable for edge autonomy.

The Architecture

Two-Lane Neuromorphic Control System:

Lane 1: Real-Time Control

Deterministic spike-based crack detection with 2.50 ms avg latency, 2.81 ms P99. Control decisions complete within 6 ms safety contract at ~106 Hz.

Lane 2: Semantic Analysis

Non-blocking YOLO inference and signature enrichment runs asynchronously without impacting Lane 1 latency guarantees.

  • Converts video into sparse spike representations via SFSVC encoding
  • Detects temporal edges through frame-delta encoding
  • Achieves 9.97× compression ratio with 89.97% bandwidth reduction
  • Delivers crack-aware control decisions under strict timing bounds
  • CPU-only processing: 2-4W vs 15-30W GPU power draw

Implementation: Private C++ SDK with SIMD AVX2 optimization and lock-free queues.

Control Output

Per-frame decision system

Crack Width Estimation

Pixel-based width calculated from spike band geometry

Severity Classification

Minor, moderate, or severe categorization

Confidence Score

Derived from spike consistency across adjacent frames

Navigation Decisions

FORWARD / CAUTION / SLOW commands based on crack severity
